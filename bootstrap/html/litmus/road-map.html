litmus road map

Litmus is currently in beta, and we’re hard at work improving it for the 1.0 release.

Here’s what’s coming down the pipe:

1. Synthetic browser

We will be revamping the HTTP and HTTPS checks to optionally use a synthetic browser. This will provide much more relevant and interesting data, including Javascript execution time and third party dependencies. We are also looking at integrating sitespeed.io to provide performance best practices.

2. Real User Monitoring

Litmus is great for providing a noise-free signal from backbone locations around the world. But what about real users on actual last mile or mobile networks? Litmus will soon measure response times from customers actual browsers and networks, allowing for a complimentary perspective.

3. Alerting

The beta release of Litmus brings a proof of concept alerting engine within Grafana that is powered by Bosun and currently purpose built for the Litmus use-case. Currently, alerting is only available for downtime and not slowdowns. We plan on improving this alerting engine and bringing it into upstream Grafana.

4. Turnkey Grafana

We want to make it easier for people who have downloaded Grafana to use the data that Litmus provides. Litmus should be a way that people can get started with Grafana easily: go from zero to gorgeously data in less than a minute.
